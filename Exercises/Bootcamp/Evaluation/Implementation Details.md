# Implementation Details

## Centralização de Dados

### Objetivo
Ter toda a informação de avaliação num local centralizado com múltiplas perspectivas.

### Visualizações Necessárias

#### Por Aluno
- Histórico completo de progresso
- Assiduidade
- Exercícios completados
- Feedback recebido
- One-on-ones
- Checkpoints

#### Por Exercício
- Taxa de conclusão
- Tempo médio
- Dificuldade percebida
- Erros comuns

#### Por Critério
- Clarity: média da turma
- Technical understanding: evolução
- Code readability: distribuição

#### Por Turma/Intake
- Comparação entre turmas
- Evolução ao longo do tempo
- Identificação de padrões

---

## Ferramentas Tecnológicas

### Opção 1: Notion
**Vantagens:**
- Bases de dados relacionadas
- Templates personalizáveis
- Colaboração em tempo real
- Visualizações múltiplas

**Desvantagens:**
- Custo para equipas grandes
- Curva de aprendizagem

### Opção 2: Google Drive + Sheets
**Vantagens:**
- Baixo custo
- Familiaridade
- Fácil partilha

**Desvantagens:**
- Menos estruturado
- Dados dispersos

### Opção 3: Airtable
**Vantagens:**
- Muito flexível
- Visualizações poderosas
- Automações

**Desvantagens:**
- Custo
- Complexidade

### Opção 4: Sistema Próprio
**Vantagens:**
- Totalmente personalizado
- Integração com outras ferramentas

**Desvantagens:**
- Tempo de desenvolvimento
- Manutenção

## Considerações Importantes

### Mínima Inércia
> "How can we do this with the smallest amount of inertia and entropy?"

**Estratégias:**
- Integrar no workflow existente
- Automatizar quando possível
- Templates pré-preenchidos
- Lembretes automáticos

### Resistência à Mudança
**Antecipar:**
- Professores podem ver como burocracia
- Alunos podem sentir-se vigiados
- Tempo extra necessário

**Mitigação:**
- Explicar benefícios
- Mostrar valor prático
- Começar pequeno
- Iterar baseado em feedback
