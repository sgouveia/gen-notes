# Módulo IV - Engenharia de Software (Semanas 8-9)

## Visão Geral

**Duração:** 2 semanas  
**Foco:** Padrões e práticas industriais  
**Objetivo:** Construir software complexo como as grandes empresas

## Conteúdo Programático

### 1. Software Engineering
- Ciclo de vida do software
- Metodologias ágeis (Scrum)
- Git e version control
- Code reviews
- Documentação

### 2. Architectural Patterns
- MVC (Model-View-Controller)
- Layered architecture
- Microservices (conceitos)
- Design patterns:
  - Singleton
  - Factory
  - Observer
  - Strategy

### 3. Java Environment Tools
- Maven/Gradle
- IDEs avançadas
- Debugging tools
- Profiling
- Build automation

### 4. Testing
- Unit testing (JUnit)
- Integration testing
- TDD (Test-Driven Development)
- Mocking
- Code coverage

### 5. Relational Databases
- Modelo relacional
- SQL avançado
- Normalização
- Índices e otimização
- JDBC
- Transactions

## Competências Desenvolvidas

Ao final deste módulo, os alunos serão capazes de:
- [ ] Aplicar padrões arquiteturais
- [ ] Usar ferramentas profissionais
- [ ] Escrever código testável
- [ ] Modelar bases de dados
- [ ] Trabalhar em equipa com Git
- [ ] Seguir boas práticas industriais

## Projetos Práticos

- Aplicação completa com MVC
- Sistema com base de dados
- Testes automatizados
- Pipeline de CI/CD básica

## A Importância das Bases de Dados

> "A forma de aceder a aplicações muda constantemente, mas os dados e o acesso a eles mantém-se e é o valor mais importante para as empresas."

- Tecnologias conservadoras
- Poucas alterações desde os anos 60
- Base essencial do software
